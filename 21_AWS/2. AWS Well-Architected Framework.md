# 설계 원칙 및 렌즈

- **설계 원칙**은 워크로드를 설계할 때 고려해야 하는 개념과 사고방식으로 6가지 핵심 요소로 구성됨

- **렌즈**는 모범 사례와 비교하여 아키텍처를 일관되게 측정하고 개선이 필요한 영역을 식별할 수 있는 방법을 제공
	- 워크로드가 정의되면 AWS Well-Architected Framework 렌즈가 자동으로 적용됨
	- 각 렌즈에는 고유한 질문, 모범 사례, 참고 사항, 개선 계획이 있음

	***워크로드(Workload)**: 클라우드 환경에서 하나의 서비스를 수행하기 위해 유기적으로 결합된 컴퓨팅 자원(EC2, S3, RDS 등), 네트워크, 애플리케이션 코드 및 데이터 전체의 집합체*

# 6가지 핵심 요소

![[../80_Assets/Pasted image 20260704013757.png]]

- **운영 우수성** - 시스템을 실행 및 모니터링하여 비즈니스 가치를 제공하고 지속적으로 지원 프로세스 및 절차를 개선하는 능력

- **신뢰성** - 의도한 기능을 원하는 시점에 정확하고 일관되게 수행하는 워크로드의 능력 (장애 복구, 컴퓨팅 리소스 동적 확보 등)

- **보안** - 위험 평가 및 완화 전략을 통해 정보, 시스템, 자산을 보호하는 동시에 비즈니스 가치를 제공하는 능력

- **성능 효율성** - 컴퓨팅 리소스를 효율적으로 사용하여 시스템 요구 사항을 충족하고, 수요 변화와 기술 발전에 따라 이러한 효율성을 유지하는 능력

- **비용 최적화** - 가장 낮은 가격으로 비즈니스 가치를 제공하도록 시스템을 실행하는 능력

- **지속 가능성** - 워크로드의 모든 구성 요소에 걸쳐 에너지 소비를 줄이고 효율성을 향상 (환경에 미치는 영향 최소화)

# AWS Well-Architected Tool

- 최신 AWS 아키텍처 모범 사례와 비교하여 워크로드 상태를 검토 및 측정하는 도구
- AWS Well-Architected Framework의 6가지 핵심 요소에 기반한 권장 사항 제공

![[../80_Assets/Pasted image 20260702183935.png]]

### 주요 기능

1. **워크로드** - 비즈니스 가치를 제공하는 구성 요소 집합을 식별. 워크로드의 예로는 전자상거래 웹 사이트, 모바일 앱용 백엔드, 분석 플랫폼 등

2. **마일스톤** - 설계부터 테스트, 가동, 프로덕션에 이르는 제품 수명 주기 동안 진화하는 아키텍처의 주요 변경 사항을 표시

3. **렌즈** - 모범 사례와 비교하여 아키텍처를 일관되게 측정하고 개선이 필요한 영역을 식별할 수 있는 방법을 제공

4. **고위험 문제(HRI)** - 비즈니스에 상당한 부정적 영향을 미칠 수 있다고 AWS가 판단한 아키텍처 및 운영상 선택

5. **중간 위험 문제(MRI)** - AWS가 판단하기에 비즈니스에 부정적 영향을 미칠 수 있지만 심각도는 HRI보다 낮은 아키텍처 및 운영상 선택

### 워크로드 검토

- AWS WA Tool은 모범 사례를 기준으로 애플리케이션 및 워크로드 상태를 검토하고, 개선 기회를 식별하고, 시간 경과에 따른 진행 상황을 추적하는 데 도움이 되도록 설계

![[../80_Assets/Pasted image 20260702201627.png]]

### 성공 측정

- **(운영 우수성)**
  평균 수리시간(MTTR), 평균 장애 간격(MTBF), 배포 빈도 및 성공률, 문제를 탐지하여 해결하는 데 걸리는 시간, 변경 관리 성공률, 자동화 적용 범위

- **(보안)**
  보안 인시던트 발생률, 시스템 패치 소요 시간, 규정 준수 감사 결과, 보안 모범 사례를 준수하는 리소스의 비율, 보안 조사 결과 수, 보안 이벤트 대응 시간

- **(신뢰성)**
  시스템 가용성 백분율, 복구 시간 목표(RTO), 복구 시점 목표(RPO), 오류율, 장애 조치 성공 횟수, 서비스 수준 계약(SLA)상의 규정 준수 여부

- **(성능 효율성)**
  리소스 사용률, 응답 시간, 지연 시간 지표, 처리량, 시스템 확장성 지표, 캐시 적중률

- **(비용 최적화)**
  작업 단위당 비용, 비용 대비 리소스 사용률, 낭비되는 비용 파악 및 절감, 예약 인스턴스 적용 범위, 비용 할당 정확도, 클라우드 투자에 대한 ROI

- **(지속 가능성)**
  탄소 발자국 지표, 리소스 효율성, 하드웨어 수명 주기 관리, 재생 에너지 사용, 환경 영향 감소, 지속 가능한 아키텍처 관행
