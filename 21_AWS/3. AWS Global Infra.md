- AWS 글로벌 클라우드 인프라는 전 세계 데이터 센터를 통해 완전한 기능을 갖춘 200가지가 넘는 서비스를 제공
- AWS에서 물리적 인프라는 Region(리전) 및 Availability Zones(가용 영역)의 형태로 AWS 글로벌 인프라를 구성

# AWS Infrastructure

AWS 서비스를 실행할 수 있는 물리적 위치

- **Data centers**
	- 실제 서버를 호스팅하는 비공개 시설
	
- **Availability Zones(AZ, 가용 영역)**
	- AWS 리소스를 가동하는 최소 단위
	- 특정 지리적 위치 내의 격리된 여러 영역
	- 여러 가용 영역에 인스턴스를 배포하고 인스턴스 1개에 장애가 발생하는 경우 다른 가용 영역의 인스턴스가 요청을 처리할 수 있도록 애플리케이션을 설계할 수 있음
	- 데이터 센터의 클러스터
	
- **Regions(리전)**
	- 1개의 지리적 영역 내에서 격리되고 물리적으로 분리된 여러 개의 가용 영역으로 구성, 이를 통해 가장 강력한 내결함성 및 안정성을 달성
	- 각 리전은 완전히 독립적
	- AZ의 클러스터
	- 해당 리전의 위치와 최종 사용자 간의 지연 시간, 해당 리전의 리소스 비용, 거버넌스, 해당 리전의 특정 서비스 사용가능 여부를 고려하여 리전 선택

	```
	us-east-1 = AWS 리전의 이름
	us-east-1a = 뒤에 나오는 문자는 가용영역(AZ)
	서비스를 AZ에 배포할 때가 있고, 리전에 배포할 때도 있음(서비스 범위 수준에 따라)
	```
	
- **AWS Local Zones(로컬 영역)**
	- 사용할 수 있는 AWS 서비스 수가 제한된 특정 위치
	- 컴퓨팅, 스토리지 서비스와 같은 엄선된 AWS 서비스가 더 많은 최종 사용자에게 더 가까운 위치에서 제공
	- 최종 사용자에게 10밀리초 미만의 지연 시간이 필요한 수요가 많은 애플리케이션에 사용
	- 사용자와 호스트 위치 사이의 지연 시간이 짧아야 하는 워크로드에 적합
	
- **Edge locations(엣지 로케이션)**
	- AWS 서비스 요청자에 가장 가까이 있는 지점
	- 전 세계 주요 도시에 위치
	- 요청을 수신하고 더 빠른 전송을 위해 콘텐츠 사본을 캐시

# 네트워킹 기본 사항

- IP 주소는 네트워크 내 리소스의 위치를 식별
- 이 주소는 2개의 부분(네트워크, 호스트)으로 구성되며, 한 부분은 네트워크를 식별하고 다른 부분은 네트워크 내의 호스트를 식별

- **IPv4 주소** - 32비트 주소로, 8비트(옥텟) 집합 4개로 그룹화
- **IPv6 주소** - 128비트 주소로, 4개의 16진수 자릿수로 구성된 8개 그룹

![[../80_Assets/Pasted image 20260706161306.png]]

- 슬래시(/) 숫자가 클수록 네트워크는 작음

### CIDR (Classless Inter-Domain Routing) 표기법

- 숫자와 점을 사용하는 표기법으로 네트워크를 식별, 슬래시 표기법으로 서브넷 마스크를 지정
- 서브넷 마스크가 /16인 경우 첫 16비트(옥텟 2개)가 네트워크 식별용으로 예약, 나머지 16비트가 호스트 식별에 사용
- AWS에서 /28(16개 IP 주소) ~ /16(65,536개 IP 주소) 사이에서 블록 크기를 지정
- Amazon VPC는 IPv4 및 IPv6 주소 지정을 지원하며, 이들 IP에 대해 상이한 CIDR 블록 크기 제한이 있음
- 기본적으로 모든 VPC와 서브넷에는 IPv4 CIDR 블록이 있어야 함

![[../80_Assets/Pasted image 20260706184807.png]]

### 서브넷 마스크

- 호스트가 로컬 서브넷에 있는지 확인하는 데 사용되는 32비트 숫자
- 호스트 비트를 모두 0으로 설정하고 네트워크 비트를 모두 1로 설정하여 생성

# Amazon Virtual Private Cloud (VPC)

- Amazon VPC는 클라우드의 네트워크 환경, AWS 계정 전용 가상 네트워크
- 사용자가 정의한 가상 네트워크 안에서 AWS 리소스를 시작할 수 있음
- VPC는 AWS 리전 중 1개에 배포되며, 해당 리전 내의 모든 가용 영역에서 리소스를 호스팅할 수 있음
- IPv4와 IPv6를 모두 사용하여 리소스와 애플리케이션에 안전하고 쉽게 액세스
