# VPC (Virtual Private Cloud)

- **Amazon VPC**는 단일 AWS 리전과 연결된 가상 네트워크 (VPC는 둘 이상의 리전에 상주할 수 없음)
- **서브넷**은 1개의 가용 영역에만 상주할 수 있음 (VPC는 한 리전 내에서 둘 이상의 가용 영역에 걸쳐 있을 수 있음)
- **인터넷 게이트웨이**를 통해 VPC 리소스가 인터넷에 도달할 수 있음
- **라우팅 테이블**은 Amazon VPC 내로 들어오고 나오며 Amazon VPC 내에서 이동하는 트래픽의 라우팅을 제어
- AWS는 Amazon VPC와 온프레미스 간의 연결을 용이하게 하는 하이브리드 클라우드 구성을 지원
- AWS 계정에는 **기본 Amazon VPC**와 **사용자 지정 Amazon VPC**, 2가지 유형의 VPC가 있음

### 기본 Amazon VPC

- AWS 계정을 생성하면 지원되는 각 AWS 리전에서 기본 Amazon VPC가 생성, 고객은 AWS Cloud 내에서 환경을 즉시 구축할 수 있음
- 기본 Amazon VPC를 사용하면 리소스 배포를 즉시 시작할 수 있으며 네트워크 내부 구조에 대해 생각할 필요가 없음
- 각 기본 Amazon VPC는 지원되는 리전 내의 각 가용 영역에 퍼블릭 서브넷을 생성
- AWS는 모든 트래픽을 허용하는 구성을 설정하므로 기본적으로 개인 정보 보호 및 격리 장치가 없음

- 복원성을 높이기 위해 기본 Amazon VPC는 가용 영역 전체에서 서브넷으로 자동 분할
	- 기본 Amazon VPC는 리전당 하나만 허용
	- 각 기본 Amazon VPC는 해당 리전의 각 가용 영역에 하나의 서브넷을 갖도록 구성
	- 각 서브넷은 Amazon VPC CIDR의 IP 주소 범위의 일부를 사용
	- 각 서브넷의 IP 주소 범위는 다른 서브넷의 IP 주소 범위에 대해 고유해야 하며 겹칠 수 없음

- 기본 Amazon VPC는 처음에 /16 IPv4 CIDR 블록, 리전의 가용 영역에 대한 /20 기본 서브넷, 인터넷 게이트웨이, 인터넷 게이트웨이에 대한 기본 라우팅, 기본 보안 그룹, 기본 네트워크 ACL 및 기본 DHCP(Dynamic Host Configuration Protocol) 옵션 집합으로 구성

### 사용자 지정 Amazon VPC

- 기본 Amazon VPC와 달리 사용자 지정 Amazon VPC의 각 구성 요소는 생성할 때 명시적으로 정의해야 함. 명시적 구성 없이는 아무것도 허용되지 않음
- IPv4 및 IPv6 지원, VPC에 대한 CIDR 블록 등의 일부 결정은 나중에 수정할 수 없음
- 서브넷, 라우팅, VPC 엔드포인트와 같은 VPC의 다른 기능은 필요에 따라 수정할 수 있음

- 기본 Amazon VPC와 유사한 사용자 지정 Amazon VPC는 논리적으로 격리된 가상 네트워크를 제공하며, 이 네트워크는 Amazon VPC가 생성되는 리전에서 지원하는 리소스와 서비스의 배포를 지원
	- 각 서브넷을 정의하여 지원되는 IP 주소 범위 선택
	- 네트워크 내부 트래픽 흐름 관리
	- 라우팅 테이블 및 네트워크 게이트웨이를 통해 트래픽이 VPC로 들어오고 나가는 방식 관리

- 사용자 지정 Amazon VPC의 경우 기본 또는 전용 테넌시를 선택할 수 있음
	- 기본 테넌시는 다른 사용자와 공유되는 공유 하드웨어에 프로비저닝
	- 전용 테넌시는 사용자의 전용 하드웨어 (전용 테넌시는 더 많은 비용)

### 기타 사항

- 기본적으로 DNS는 Amazon VPC에 의해 처리. 그러나 Amazon Route 53을 사용하여 VPC 내에서 자체 DNS를 생성할 수도 있음
- 기본적으로 Amazon VPC의 모든 서브넷은 서로 액세스할 수 있음. 네트워크 ACL을 사용하여 서브넷으로 들어오고 나가는 트래픽을 제한할 수 있음
