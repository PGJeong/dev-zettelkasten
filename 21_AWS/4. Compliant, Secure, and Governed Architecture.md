# 전송 중인 데이터 보호 (인증서)

- TLS 인증서를 사용하면 웹 브라우저에서 웹 사이트에 대한 암호화된 네트워크 연결을 식별하고 설정할 수 있음
- 인증서는 PKI(퍼블릭 키 인프라)라는 암호화 시스템에 사용

- 요청을 생성할 때 프라이빗 키로 디지털 서명을 생성하고 해당 서명을 인증서와 함께 요청에 포함
- AWS는 인증서에 있는 퍼블릭 키로 서명을 해독하여 사용자가 발신자인지 확인
- 또한 전송한 인증서가 AWS에 업로드된 인증서와 일치하는지도 검증

### AWS Certificate Manager (ACM)

- ACM은 퍼블릭 및 프라이빗 인증서를 모두 관리하는 단일 인터페이스를 제공
- 내부 서버 및 사물 인터넷(IoT) 디바이스와 같은 온프레미스 리소스에 대한 퍼블릭 및 프라이빗 인증서를 생성하고 배포
- Elastic Load Balancing(ELB), Amazon CloudFront 배포, Amazon API Gateway 엔드포인트와 같은 AWS 리소스에 프라이빗 또는 퍼블릭 인증서를 배포
- 프라이빗 인증서를 보호하고 관리. ACM은 잘못 구성되거나 만료된 인증서로 인한 가동 중지 시간을 최소화하는 데 도움이 될 수 있음

### AWS Private Certificate Authority

- 프라이빗 인증서의 수명 주기를 쉽고 안전하게 관리하는 데 도움이 되는 관리형 프라이빗 CA 서비스
- 퍼블릭 및 프라이빗 인증서를 중앙에서 관리하고 ACM이 사용자를 대신하여 프라이빗 인증서를 자동으로 갱신하도록 함
- AWS에서 조직의 전체 프라이빗 인증서 인프라를 호스팅하고 관리할 수 있음

# 애블리케이션 보호 (AWS Shield, WAF)

- AWS WAF(Web Application Firewall)와 AWS Shield가 연동되어 웹 애플리케이션을 포괄적으로 보호하는 방식
- WAF에서는 HTTP/HTTPS 트래픽을 필터링 및 모니터링하는 사용자 지정 가능 보안 규칙을 생성할 수 있으며, Shield에서는 DDoS 공격으로부터 애플리케이션을 보호할 수 있음

### AWS Shield

- 모든 AWS 고객은 추가 비용 없이 AWS Shield Standard의 자동 보호를 받을 수 있음

![[../80_Assets/Pasted image 20260714230912.png]]
- AWS Shield Standard는 웹 사이트나 애플리케이션을 대상으로 가장 일반적이고 빈번하게 발생하는 네트워크 및 전송 계층 DDoS 공격을 방어
- Shield Standard는 이상 탐지를 제공하기 위해 트래픽의 기준을 설정하고 독점 패킷 필터링을 사용하여 의심스러운 트래픽이 서버에 도달하는 것을 방지
- AWS 전체에 걸쳐 기준 설정 및 이상 탐지

### AWS WAF

- 계층 7에서 작동하는 AWS WAF는 네트워크 패턴만이 아닌 콘텐츠를 기반으로 트래픽을 검사 및 필터링

![[../80_Assets/Pasted image 20260714231138.png]]
- 일련의 규칙을 통해 요청을 처리하며 각 규칙은 웹 ACL 용량 단위(WCU)를 사용
- 기본 제공되는 WCU 1,500개로는 기본 규칙 100개, 복합 규칙 50개 정도를 구현할 수 있음
- 이러한 구현은 기본 검사용, 자세한 검사용 등으로 여러 보안 검문소를 설치하는 방식과 비슷
- 이 용량을 활용하면 응답 시간을 밀리초 단위 이하로 유지하면서 자세한 규칙 세트를 작성할 수 있음

- AWS WAF를 사용하여 다음 동작 중 하나를 선택할 수 있음
	- 지정한 요청을 제외한 모든 요청을 허용
	- 지정한 요청을 제외한 모든 요청을 차단
	- 지정한 속성과 일치하는 요청 수를 계산

- WAF를 사용하여 생성할 수 있는 여러 규칙 유형
	- **AWS 관리형 규칙** - Marketplace에서 제공되는 규칙을 사용하면 전문 보안 공급업체의 전문 지식을 활용할 수 있음
	- **관리형 AWS Marketplace 규칙** - AWS Marketplace 관리형 규칙 그룹은 AWS Marketplace에서 AWS Marketplace 콘솔을 통해 구독 방식으로 제공
	- **고객 관리형 규칙** - 프로젝트 관리자가 요청한 국가별 필터와 사용자 지정 보안 요구 사항을 구현

![[../80_Assets/Pasted image 20260714232126.png]]
※ **웹 ACL(웹 액세스 제어 목록)**: 웹 리소스로 들어오는 HTTP/HTTPS 요청을 세밀하게 제어하고 필터링하는 보안 규칙의 집합

# 모니터링

- AWS WAF에서는 3가지 유용한 모니터링 도구를 활용하여 웹 보안 태세를 확인할 수 있음

- **Amazon CloudWatch** - Amazon CloudWatch를 사용하여 웹 요청, 웹 ACL 및 규칙을 모니터링, AWS WAF 및 AWS Shield Advanced에서 원시 데이터를 수집한 후 판독이 가능하고, 준실시간 지표로 처리
- **AWS WAF 콘솔** - AWS WAF 콘솔에서 요청 샘플링을 활성화한 경우 연결된 리소스가 검사를 위해 AWS WAF에 전달한 요청의 샘플을 볼 수 있음
- **전체 로그** - WAF의 전체 로깅 기능을 활용하면 CloudWatch Logs, S3 버킷, Kinesis Data Firehose를 통해 필요한 모든 정보를 확인할 수 있음

# Amazon GuardDuty

- GuardDuty는 의심스러운 활동(예: 알려진 악성 IP의 정찰 시도 가능성)이 탐지되면 WAF 규칙 업데이트를 자동 트리거하여 해당 트래픽을 차단할 수 있음
- 대응이 자동으로 진행되므로 응답 시간이 몇 시간 단위에서 몇 초 단위로 대폭 단축

# AWS Secrets Manager

- Secrets Manager는 보안 암호를 더 쉽게 관리할 수 있게 해주는 AWS 서비스
- 보안 암호는 데이터베이스 자격 증명, 암호, 서드 파티 API 키 및 심지어 임의 텍스트가 될 수 있음
- Secrets Manager 콘솔, Secrets Manager 명령줄 인터페이스(CLI) 또는 Secrets Manager API 및 SDK를 사용하여 중앙에서 이러한 보안 암호를 저장하고 보안 암호에 대한 액세스를 제어할 수 있음

- 암호를 포함하여 코드에 하드코딩된 자격 증명을 Secrets Manager에 대한 API 직접 호출로 대체하여 프로그래밍 방식으로 보안 암호를 검색
- 지정한 일정에 따라 보안 암호를 자동으로 교체하도록 Secrets Manager를 구성할 수 있음

# AWS KMS (Key Management Service)

- AWS KMS는 데이터 암호화에 사용되는 키를 생성하고 제어할 수 있는 관리형 서비스
- 다양한 키 기능을 수행하여 1개의 중앙 위치에서 키를 관리할 수 있음
- 더 나은 관리를 위해 고유한 별칭과 설명을 사용하여 KMS 키를 생성할 수 있음
- 일정에 따라 자동으로 키를 교체하고 다른 사람이 사용할 수 없도록 키를 비활성화하거나 삭제할 수 있음
- 감사의 경우 CloudTrail을 분석하여 키를 누가 사용했는지, 언제 사용했는지, 어떻게 사용했는지 확인할 수 있음

### 봉투 암호화

- AWS KMS는 봉투 암호화를 사용하여 데이터를 보호
- 봉투 암호화는 2가지 다른 키를 사용하는 데이터 암호화에 최적화된 방법

- 데이터 키는 AWS 서비스에서 생성되어 각 데이터 또는 리소스를 암호화하는 데 사용
- 그러면 데이터 키는 사용자가 정의한 다른 KMS 키인 루트 키로 암호화
- AWS KMS는 데이터 키를 암호화하는 데 사용된 KMS 키를 추적
- AWS 서비스는 암호화된 데이터 키를 영구 스토리지에 저장

![[../80_Assets/Pasted image 20260715192848.png]]

### 다중 리전 키

- AWS KMS는 서로 바꿔 사용할 수 있는 여러 리전의 AWS KMS 키인 다중 리전 키를 지원
- 이는 마치 여러 리전에 동일한 키가 있는 것과 같음
- 다중 리전 키(MRK)의 각 세트는 동일한 키 구성 요소와 키 ID를 가짐
- 리전 1곳의 데이터를 암호화하고 다른 리전에서 해독할 수 있음
